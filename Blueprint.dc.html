0. สถานะปัจจุบัน — สิ่งที่สร้างเสร็จแล้วก่อนเริ่ม Blueprint นี้

หน้าเว็บที่สร้างเสร็จแล้ว: Home, Search Results, Property Details, Sell, About, Contact, Admin Login, Admin Dashboard, Owners, Property Map, AI Quick Add, Site Content, Agent Signup, Agent Approvals, Agent Profile (สาธารณะ), Lister Dashboard, Lister Billing, Team, Staff Signup, Advertise, AI Concierge — พร้อม shared components: PropertyCard, SearchFilters, LanguageSwitcher, ContactRail

ฟีเจอร์ที่ใช้งานได้แล้วจริง:

  • ระบบ 8 ภาษาเต็มรูปแบบ ทุกหน้า+ฟอร์ม+แชท+แอดมิน
  • แชทบอท AI (Claude API ผ่าน Cloud Function) ตอบลูกค้าอัตโนมัติ + สร้างปุ่มลิงก์อัตโนมัติ
  • AI Quick Add, Favorites+Intent Scoring, Admin Dashboard สถิติครบ, Maintenance Gate
  • ระบบ self-serve เต็มรูปแบบสำหรับผู้ลงประกาศ (สมัคร/โพสต์/แก้ไข/ลบ/แชร์เว็บส่วนตัว)
  • Stripe subscription + Featured boost + แบนเนอร์ภายนอก + VIP ทั้งสองฝั่ง — ครบวงจร

1. รูปแบบธุรกิจ & ทีมงาน

  • นิติบุคคล: ทำในนามบุคคลธรรมดา/อิสระได้ก่อน ค่อยพิจารณาเปิดบริษัทเมื่อรายได้/ทีมโตพอ
  • ทีมงาน: ขยายจาก 1 คน → 2-3 คนสูงสุด สิทธิ์ 2 ระดับ Owner/Staff
  • Remote core + Local hands: Owner ดูแลจากที่ไหนก็ได้ Staff พื้นที่ถ่ายรูป+ป้อนข้อมูล

2. รายได้ 3 ทาง (+ ทาง 4 VIP)

2.0 ขอบเขตพื้นที่ให้บริการ: เฉพาะหัวหิน/ชะอำ/ปราณบุรี — ตรวจพิกัดก่อนอนุญาตโพสต์ (ยังทำผ่านการตรวจแบบ manual ในขั้น AI Quick Add เท่านั้น)

ทาง 1 — ค่าคอมมิชชัน: favorite → intent score → threshold → บอท AI เปิดบทสนทนาเจาะจง → lead แท็ก "hot"

✓ ทาง 1 เสร็จสมบูรณ์ — favorites.js, intent scoring, auto-bot, lead ใน Admin Dashboard

2.1 "ให้ทีมช่วยขาย" (Assisted Sale) — แทนที่ Agent VIP pool เดิม: ไม่มีสถานะ Agent แยกจากผู้ลงประกาศทั่วไปแล้ว ใครก็โพสต์ได้อิสระ บ้านซ้ำ/แข่งราคาเป็นเรื่องปกติของตลาด

◐ ทำไปแล้วบางส่วน — checkbox+% commission หลังบันทึกทรัพย์ ✓, ป้าย nudge ในการ์ด ✓ — ยังไม่มี: workflow tracker (รับเรื่อง→ปิดการขาย), แบนเนอร์/อีเมลแจ้งเตือนเชิงรุก

ทาง 2 — ค่าสมัครสมาชิกโพสต์เอง: บันไดสมาชิก 5 ระดับ (ฟรี/②/③/VIP เงิน/VIP ทอง — เพชรรอเปิด), Rollout Level สวิตช์เดียวคุมการเปิดเผยแต่ละระดับ

✓ Stripe subscription ครบวงจร (Checkout, webhook, payments ledger, Customer Portal)
✓ self-serve dashboard เต็มรูปแบบ (Lister Dashboard) — โพสต์/แก้ไข/ลบทรัพย์ทั้งใบ/แชร์เว็บส่วนตัว
✓ โปรไฟล์ Agent สาธารณะ (Agent Profile.dc.html) พร้อมแชทบอทของตัวเอง
✓ Founding Agents อย่างเป็นทางการ — จำกัดจำนวน + ข้อความเงื่อนไขตั้งได้จาก Site Content
✓ ล็อกอินง่าย — Google / Facebook / เบอร์โทร OTP / อีเมล+รหัสผ่าน (ต้องเปิดสวิตช์ผู้ให้บริการที่ Firebase Console ก่อนใช้จริง)
✓ จำกัดจำนวนโพสต์ตามแพ็กเกจ (tierLimit ผูกกับปุ่ม "+ เพิ่มทรัพย์ใหม่")
○ ยังไม่ได้ทำ: Facebook post AI ร่างสำนวนจริงในข้อความโพสต์ (ตอนนี้เป็นเทมเพลตประกอบข้อมูล), TikTok caption generator, pay-per-listing

ทาง 3 — โฆษณา/แบนเนอร์: slot ปรับได้, สุ่มหมุนเวียน, ราคาตามหน้า×ตำแหน่ง, ขายให้ลูกค้าภายนอกจริงผ่าน Stripe

✓ เสร็จสมบูรณ์ — Advertise.dc.html + ตัวกรอง "ทั้งหมด/ของเว็บเอง/รอชำระเงิน/จ่ายแล้ว" ใน Site Content, Featured boost, badge Featured/VIP บนการ์ดสาธารณะ

ทาง 4 — แพ็กเกจ VIP (เจ้าของบ้าน + Agent): เดิมเป็นเฟสอนาคต ตอนนี้เปิดใช้งานแกนหลักแล้ว

✓ ปุ่ม VIP เงิน/ทอง/เพชร ต่อทรัพย์ในหน้า Admin (ฝั่งเจ้าของบ้าน) ผ่าน Stripe one-time
✓ Agent VIP subscription ในหน้า Lister Billing (ฝั่ง Agent) พร้อม teaser ล่อใจ (จำนวนบ้าน/Agent ในพูลที่ยังเข้าไม่ถึง)
✓ ระบบลงทะเบียนผู้ซื้อ + รหัส Agent เฉพาะตัว (ป้องกันข้อพิพาท Procuring Cause) — คุ้มครอง 60 วัน
✓ Badge "VIP" บนการ์ดสาธารณะ + "Verified VIP" บนโปรไฟล์ Agent
○ ยังไม่ได้ทำ: หน้าจัดการข้อพิพาทสำหรับแอดมิน (audit trail), Buyer Registration ยังไม่ผูกกับ UI ฝั่งสาธารณะ (ตอนนี้อยู่ที่ Lister Dashboard ของ Agent เท่านั้น)

3. การสร้างตัวตนแบรนด์ (AI-powered)

Landing Page Agent (Agent Signup.dc.html) ✓, เมนูตัวเลือกด่วนในแชทบอท ✓ — ป้าย "Powered by AI" และ signature ท้าย Facebook post ยังไม่ยืนยันสถานะล่าสุด

4-6. การเงิน / ความปลอดภัย / SEO

การเงิน: Stripe ledger + webhook + auto-hide เมื่อจ่ายไม่ผ่าน — เสร็จสมบูรณ์

ความปลอดภัย: Firestore/Storage rules role-based ✓ deploy แล้ว — App Check, rate limiting, reCAPTCHA v3, backup อัตโนมัติ ยังไม่ได้ทำ

✓ ตรวจสอบแล้ว (16 ก.ค. 2026): ใช้ role-based + deny-by-default จริงแล้ว ไม่ใช่ rules เปิดโล่งชั่วคราวอีกต่อไป

SEO: URL ถาวร + Open Graph + JSON-LD ที่ Property Details ✓, sitemap.xml + robots.txt (หน้า static หลัก) ✓ — ยังไม่ได้ทำ: sitemap รายทรัพย์แต่ละหลัง (ต้อง prerendering), ยื่น Search Console จริง

7-10. แดชบอร์ดสถานะระบบ / แจ้งเตือน / วงจรชีวิตประกาศ / แบนเนอร์

แดชบอร์ดสถานะระบบ (3 หมวด) ✓, วงจรชีวิตประกาศ+ลบรูปอัตโนมัติ (ขั้นพื้นฐาน, client-side sweep) ✓, ระบบจัดการแบนเนอร์ Admin CRUD ✓

○ ยังไม่ได้ทำ: LINE Official Account แจ้งเตือน (ไม่เร่งด่วน), สรุปประจำวัน/รายเดือน/รายปี, Staging (Preview Channels)

11. ลำดับการสร้างจริง — สถานะปัจจุบัน

ชุด A และ B เสร็จสมบูรณ์ทั้งหมดแล้ว รวมทาง 4 (VIP) — เหลือแค่งานที่ตั้งใจเลื่อนไว้ไม่เร่งด่วน
✓
ชุด A (ทาง 1 ครบ, แบนเนอร์ครบ, Agent Signup+Approvals+Profile+Dashboard ครบ, วงจรชีวิตประกาศพื้นฐาน)
✓
ชุด B ข้อ 1-6 (Security Rules, Storage, role Owner/Staff, Stripe, Featured, แบนเนอร์ภายนอก)
✓
ชุด B ข้อ 7 — ทาง 4 VIP (แกนหลัก) — เสร็จแล้ว รวม Buyer Registration + Agent Code + Badge + Teaser
◐
งานเก็บตก: Assisted Sale workflow tracker, ตัวกรองแบนเนอร์ (ทำแล้ว) แต่หน้า Admin ยังไม่มีมุมมองแยกชัด, หน้าจัดการข้อพิพาท VIP
○
LINE Official Account แจ้งเตือน (ไม่เร่งด่วน ทำเมื่อไหร่ก็ได้)
งานหลักของ Blueprint ฉบับนี้เสร็จสมบูรณ์แล้ว — ที่เหลือเป็นงานเก็บตก/เสริมที่ไม่บล็อกการใช้งานจริง และ LINE แจ้งเตือนซึ่งตั้งใจเลื่อนไว้ไม่เร่งด่วน

12. แผนเปิดตัว (Go-to-Market)

หลักการรวม: ทุกกลุ่มใช้ฟรีก่อนเสมอ แล้วค่อยเปิดจ่ายทีละชั้นเมื่อเห็นผลจริง

  1. ปิดเงียบ (Soft launch) 2-4 สัปดาห์
  2. เปิดวงในพื้นที่ + Founding Agents ฟรี
  3. SEO/Google Business คู่ขนาน
  4. เปิดเก็บเงินทาง 2
  5. เปิดทาง 3 (โฆษณาลูกค้าภายนอก)
  6. เปิดทาง 4 (VIP ทั้งสองฝั่ง) — พร้อมใช้งานแล้วตั้งแต่ตอนนี้

14. หน้า AI Concierge (Conversational Landing Page)

✓ สร้างเสร็จแล้ว — AI Concierge.dc.html (ยังไม่ได้อัปโหลดขึ้นเว็บจริง รอตัดสินใจ)
✓ บทสนทนาต่อเนื่องข้ามหน้า (localStorage คีย์เดียว)
✓ ปุ่มย่อ/ขยายกล่องแชท ("─" ไม่ใช่ "×")

✓ เลื่อนอัตโนมัติ + ปุ่มล้างประวัติแชทมีข้อความกำกับ (ContactRail + AI Concierge)

○ ยังไม่ได้ทำ: ปุ่มแชร์ครบรูปแบบ (Web Share API), เก็บสถิติ ref code ต่อลิงก์

15-17. UX หน้า Home / กฎป้องกันการสวมประกาศ / มอบหมายงาน

✓ แบนเนอร์รูปแบบใหม่ (PropertyCard ปกติ ไม่มีป้าย Sponsored), CTA ย้ายใต้ช่องค้นหา, Hero มือถือชิดบน-ซ้าย
✓ กฎป้องกันการ "สวมประกาศ" — ล็อกฟิลด์ตัวตนหลัก (รูปปก/ราคา/ที่อยู่/ชื่อ/รหัส) เมื่อสถานะเป็น "sold"
○ ระบบมอบหมายงานตามแผนก/โซน — เตรียมแนวคิดไว้ล่วงหน้าเท่านั้น ยังไม่สร้าง UI จริง (รอทีมโตถึง 3+ คน)
