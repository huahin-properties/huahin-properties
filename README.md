# huahin.properties — อัปเดตล่าสุด (ลองใหม่อัตโนมัติเมื่อ AI ตอบข้อมูลผิดรูปแบบ)

## สาเหตุที่ยังเจอ error เดิม
รอบที่แล้วเราซ่อมแซมข้อความที่ผิดรูปแบบเล็กน้อยอัตโนมัติ (เช่น ขึ้นบรรทัดใหม่ในที่ที่ไม่ควรมี) แต่ยังมีบางครั้งที่ AI ตอบข้อมูลผิดรูปแบบมากกว่านั้น (เช่น เครื่องหมายคำพูดไม่ถูกตำแหน่ง) ที่ซ่อมแซมไม่ได้

## การแก้ไขในรอบนี้
**เพิ่มระบบลองใหม่อัตโนมัติ** — ถ้า AI ตอบข้อมูลผิดรูปแบบจนอ่านไม่ได้ ระบบจะเรียก AI ใหม่ให้อัตโนมัติสูงสุด 3 ครั้งก่อนจะแจ้ง error ให้เห็น (ปกติ AI จะไม่ตอบผิดรูปแบบซ้ำกันสองครั้งติด) ผู้ใช้ไม่ต้องกดปุ่มซ้ำเองอีกต่อไป

## วิธีใช้งาน
1. อัปโหลดไฟล์ทั้งหมดในโฟลเดอร์นี้ขึ้น repo บน GitHub ที่ระดับ root (ทับไฟล์เดิมทั้งหมด)
2. รอ GitHub Pages deploy เสร็จ (1-2 นาที) แล้ว Hard Refresh (Ctrl+Shift+R)
3. ลองกด "ให้ AI ร่างประกาศ" ใหม่

## ⚠️ ข้อควรรู้ (เหมือนเดิม)
- Firestore Security Rules เปิดกว้างชั่วคราว หมดอายุ 8 ส.ค. 2026
- ระบบล็อกอิน Admin เป็นการป้องกันเบื้องต้น ไม่ใช่ระบบความปลอดภัยระดับ production เต็มรูปแบบ
- มีการเรียกใช้ Anthropic API + Google Maps API จริง (การลองใหม่อัตโนมัติหมายถึงอาจเรียก AI มากกว่า 1 ครั้งต่อการร่าง 1 รอบ ถ้าเจอปัญหา) — เช็คยอดใช้จ่ายเป็นระยะ
