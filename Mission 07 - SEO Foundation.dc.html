✅ Step 3 — Canonical URL Strategy

✅ Step 4 — Social Meta

✅ Step 5 — Structured Data (Site-wide)

เพิ่ม JSON-LD RealEstateAgent (Organization-level, ทุกหน้า) + WebSite พร้อม SearchAction (หน้า Home เท่านั้น ชี้ไปที่ Search Results) — ยังไม่ได้ทำ BreadcrumbList (ต้องมีโครงสร้าง URL/breadcrumb UI ก่อน ปัจจุบันเว็บไซต์เป็น flat page structure ไม่มี breadcrumb UI จริง จึงไม่ใส่ schema ที่ไม่ตรงกับหน้าจอจริง — ตาม Google guideline ห้ามใส่ schema ที่ไม่ match เนื้อหาที่แสดงจริง)

✅ Step 8 — Multilingual SEO (Complete — Documentation)

ยังไม่ใส่ hreflang จริง — เหตุผล: เว็บไซต์ปัจจุบันสลับภาษาแบบ client-side state (เก็บใน state/localStorage) ไม่ใช่ URL แยกต่อภาษา — การใส่ hreflang ชี้ไปที่ URL เดียวกันซ้ำทุกภาษาผิดหลัก Google และเป็นโทษมากกว่าไม่ใส่ จึงไม่สร้างตามคำสั่ง Product Owner Revision 2

Future Enhancement — URL Routing Recommendation

Migration Plan: เปลี่ยนจาก client-side language switch เป็น URL-based routing เช่น /en/, /th/, /ru/, /zh/, /de/, /no/, /fr/, /it/ (หรือ query param ?lang= ที่มี canonical แยกต่อภาษาจริง) — ต้องปรับ routing/hosting config และ logic class ทุกหน้าให้อ่านภาษาจาก URL แทน state ล้วนๆ

SEO Impact Analysis: เมื่อมี URL แยกจริงต่อภาษา จะสามารถใส่ <link rel="alternate" hreflang="th|en|ru|zh|de|no|fr|it" href="..."> + hreflang="x-default" ได้ถูกต้อง ทำให้ Google จัดอันดับแต่ละภาษาแยกกันในผลค้นหาของประเทศนั้นๆ (เพิ่ม organic traffic จากผู้ใช้ที่ค้นด้วยภาษาอื่นนอกจาก TH/EN อย่างมีนัยสำคัญ)

เหตุผลที่ยังไม่ทำตอนนี้: เป็นการเปลี่ยนสถาปัตยกรรม routing ทั้งเว็บไซต์ กระทบทุกหน้า ทุกลิงก์ภายใน และ Firebase Hosting config — ต้องได้รับการอนุมัติจาก CEO/Product Owner ก่อน ไม่ใช่การตัดสินใจที่ควรทำเองในรอบ SEO Mission
