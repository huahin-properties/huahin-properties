0. สถานะปัจจุบัน — สิ่งที่สร้างเสร็จแล้วก่อนเริ่ม Blueprint นี้

เอกสารนี้ครอบคลุมตั้งแต่จุดเริ่มต้นโปรเจกต์จนถึงแผนอนาคตทั้งหมดในที่เดียว หมวดนี้คือ "สร้างเสร็จแล้ว" ส่วนหมวด 11 (ลำดับการสร้างจริง) คือสิ่งที่ยังไม่ได้ทำนับจากนี้เป็นต้นไป

หน้าเว็บที่สร้างเสร็จแล้ว: Home, Search Results, Property Details, Sell, About, Contact, Admin Login, Admin Dashboard, Owners, Property Map, AI Quick Add, Site Content — พร้อม shared components: PropertyCard, SearchFilters, LanguageSwitcher, ContactRail

ฟีเจอร์ที่ใช้งานได้แล้วจริง:

  • ระบบ 8 ภาษาเต็มรูปแบบ ทุกหน้า+ฟอร์ม+แชท+แอดมิน
  • แชทบอท AI (Claude API ผ่าน Cloud Function) ตอบลูกค้าอัตโนมัติ + สร้างปุ่มลิงก์อัตโนมัติ
  • AI Quick Add: ถ่ายรูปหลายรูป, พูดบอกข้อมูล, พิกัด Google Maps, ร่างประกาศ+คำนวณระยะทาง+โพสต์ Facebook อัตโนมัติ
  • Admin Dashboard: สถิติครบ + tracker สัญญาเช่าใกล้หมดอายุ
  • โหมดปิดปรับปรุงเว็บไซต์ (Maintenance Gate)
  • ระบบรหัสผ่านแอดมิน + data.js ตัวอย่าง 9+ รายการ

งานถัดจากนี้เริ่มที่หมวด 11 ข้อ 1 เป็นต้นไป — ทุกอย่างก่อนหน้านั้นถือว่าเสร็จแล้ว

1. รูปแบบธุรกิจ & ทีมงาน

  • นิติบุคคล: ทำในนามบุคคลธรรมดา/อิสระได้ ไม่ต้องเปิดบริษัทตอนนี้ Stripe รับเงินให้บุคคลธรรมดาได้ปกติ พอรายได้/ทีมโตพอค่อยพิจารณาเปิดบริษัท
  • ทีมงาน: ขยายจาก 1 คน → 2-3 คนสูงสุด สิทธิ์ 2 ระดับพอ: Owner (เห็น/แก้ทุกอย่าง+การเงิน) / Staff (จัดการประกาศ/lead/แชท แตะการเงิน-ตั้งค่าไซต์ไม่ได้)
  • โมเดลการทำงาน — Remote core + Local hands: ระบบ cloud-native ทั้งหมด จัดการได้จากทุกที่ Owner ดูแลเว็บ/การเงินจากที่ไหนก็ได้ Staff พื้นที่ถ่ายรูป+ป้อนข้อมูลผ่าน AI Quick Add หน้างาน

2. รายได้ 3 ทาง

2.0 ขอบเขตพื้นที่ให้บริการ: เฉพาะ 3 อำเภอ หัวหิน/ชะอำ/ปราณบุรี ตรวจสอบพิกัด Google Maps ก่อนอนุญาตโพสต์ทุกครั้ง — ปฏิเสธทันทีถ้าอยู่นอกเขต (รักษาความน่าเชื่อถือ + SEO เฉพาะทาง)

ทาง 1 — ค่าคอมมิชชัน (ทำอยู่แล้ว): กด favorite → intent score เงียบๆ → ถึง threshold → บอท AI เปิดบทสนทนาเจาะจง → เก็บ requirement → บันทึกเป็น lead แท็ก "hot"

ทาง 2 — ค่าสมัครสมาชิกโพสต์เอง (Agent/เจ้าของบ้าน, Stripe): จุดขาย AI ช่วยร่างประกาศ 8 ภาษา, ผู้ช่วย AI 24 ชม., ระบบ lead อัตโนมัติ แพ็กเกจ Basic/Pro/Agency + pay-per-listing, โปรไฟล์ Agent สาธารณะ, ระยะฟรี "Founding Agents" ช่วงเปิดตัว (ประกาศเงื่อนไขชัด, ตั้ง milestone เปลี่ยนผ่าน, จำกัดจำนวน)

Flow: สมัคร → เลือกแพ็กเกจ/ฟรี → Stripe Checkout → approve → โพสต์ผ่าน self-serve → จัดการผ่าน Stripe Customer Portal

Facebook post AI ร่าง + ลิงก์ backlink กลับหน้าประกาศเฉพาะ (traffic + SEO สองต่อ) — TikTok caption แยกดีไซน์ (ไม่รองรับลิงก์คลิกได้ ใช้รหัสประกาศ+bio link แทน) — ข้อมูลติดต่อเติมอัตโนมัติจากโปรไฟล์ (แก้รายประกาศได้) — ทุกประกาศมีลิงก์เฉพาะ+ปุ่ม Copy Link ให้ลูกค้ากระจายเอง

ล็อกอิน Agent/เจ้าของบ้านแบบง่าย: Firebase Authentication เดียวกับแอดมิน รองรับ Google / Facebook / เบอร์โทร OTP / อีเมล+รหัสผ่าน ผูกบัญชีเดียวกันทุกช่องทาง

ทาง 3 — โฆษณา/แบนเนอร์: เติมเนื้อหาเองช่วงแรกกันหน้าว่าง, slot จำนวนปรับได้ต่อตำแหน่ง, สุ่มหมุนเวียนถ้าคนจ่ายเกิน slot, คลิกเข้าหน้าโฆษณาในเว็บเอง (ไม่ออกนอกเว็บ), ราคาตามหน้า×ตำแหน่ง×ขนาด, Featured Listing boost ต่อยอด subscription เดียวกัน

ทาง 4 — Agent VIP / Curated High-Commission Pool (เฟสอนาคต หลังทาง 2 มั่นคง): รายได้ 2 ฝั่งเก็บแน่นอน 100% — ฝั่งเจ้าของบ้านจ่ายแพ็กเกจ VIP (เงิน/ทอง/เพชร) เพื่อเข้าพูล Agent VIP เห็น/ติดต่อมากขึ้น, ฝั่ง Agent จ่ายค่าสมาชิก VIP เพื่อเข้าถึงบ้านคุณภาพดี — ค่าคอมระหว่าง Agent กับเจ้าของบ้านไม่เกี่ยวกับแพลตฟอร์มเลย แพลตฟอร์มมีสิทธิ์ขายเอง/ใช้ Agent ในสังกัดแบ่งค่าคอมได้ด้วย ระดับ VIP ขยายได้ไม่จำกัด มี teaser ล่อใจ + ตัวเลขสต๊อกโชว์สองฝั่ง สัญญา 2 ฉบับเชื่อมกัน (แพลตฟอร์ม↔เจ้าของบ้าน, Agent↔เจ้าของบ้าน) กติกา Procuring Cause + Buyer Registration ป้องกันข้อพิพาท Agent Code ป้องกันสวมสิทธิ์ เบอร์ติดต่อเจ้าของบ้านแสดงปกติไม่ซ่อน

3. การสร้างตัวตนแบรนด์ (AI-powered)

Landing Page สำหรับ Agent/เจ้าของบ้านแยกเฉพาะ, ทดลองคุยกับแชทบอทก่อนสมัคร, signature ท้าย Facebook post, ป้าย "Powered by AI" บนเว็บสาธารณะ

3.1 เมนูตัวเลือกด่วนในแชทบอท: หลังทักทาย ส่งปุ่มเลือกทันที (ฝากขาย/ฝากเช่า/สมัคร Agent/หาบ้าน/คำถามอื่น) พาไปสู่ flow ที่ถูกต้องทันที ปรับได้จาก Site Content

4. ระบบการเงิน — ให้ Stripe ทำงานหนักแทนเรา

Ledger จาก webhook ทุกครั้งที่ตัดเงินสำเร็จ, แจ้งเตือนก่อนหมดอายุผ่าน Stripe Billing, หยุดอัตโนมัติถ้าจ่ายไม่ผ่าน (ซ่อนประกาศ ไม่ลบข้อมูล) กลับมาโชว์ทันทีเมื่อจ่ายสำเร็จ — สร้างจริงแค่ 1 หน้า self-serve (Stripe Customer Portal) + 1 Cloud Function รับ webhook

5. ความปลอดภัย & ป้องกันสแปม/แฮก

  1. Firestore Security Rules แบบ role-based
  2. Firebase App Check — กันบอทยิง request ตรง (ฟรี)
  3. Rate limiting ที่ Cloud Functions
  4. reCAPTCHA v3 แบบเงียบ
  5. Stripe webhook signature verification
  6. สำรองข้อมูลอัตโนมัติ (Firestore scheduled export → Storage)
  7. บัญชีแอดมิน — รหัสผ่านแข็งแรง + พิจารณา 2FA

6. SEO (ความสำคัญสูงสุด)

  1. URL ถาวรต่อประกาศ
  2. Meta title/description เฉพาะหน้า + Open Graph
  3. Structured data (schema.org) เป็น JSON-LD ต่อประกาศ
  4. Sitemap.xml อัตโนมัติ + Google Search Console
  5. Prerendering/dynamic rendering สำหรับ crawler
  6. ความเร็วโหลดหน้า (Core Web Vitals)
  7. ชื่อไฟล์รูป/alt text SEO-friendly
  8. Google Business Profile + Search Console ตั้งแต่ soft launch

7. แดชบอร์ด "สถานะระบบ & สิ่งที่ต้องทำ"

การ์ดแจ้งเตือนสีตามความเร่งด่วน แบ่ง 3 หมวด: A. เช็คอัตโนมัติ (โควตา/subscription/สัญญาเช่า/lead ใหม่) B. ปฏิทินความปลอดภัย (Firestore rules หมดอายุ, checklist deploy, รีวิว API key) C. บริการภายนอกที่แอดมินกรอกเอง (โดเมน ฯลฯ เตือนล่วงหน้า 30/14/3/1 วัน)

8. การแจ้งเตือน & กระบวนการอัปเดตเว็บอย่างปลอดภัย

8.1 LINE Official Account (ทำภายหลัง ไม่เร่งด่วน): แจ้งเฉพาะเรื่องพลาดไม่ได้ (ชำระเงิน, hot lead, ฟอร์มติดต่อ, Agent สมัครใหม่, สัญญาเช่าใกล้หมดอายุ, ปัญหาระบบ) กิจกรรมเบาๆ ดูในแดชบอร์ดพอ

8.2 Staging ก่อนขึ้นจริง: จุด A (Production) นิ่งเสมอ, จุด B (Staging) ทดลองก่อน ผ่าน Firebase Hosting Preview Channels — ทดสอบลิงก์ทดลองก่อน แล้วค่อย "เลื่อนขึ้น" เป็นเว็บจริง เว็บจริงไม่มีวันพังกลางอากาศ

8.3 Integration Registry: Firebase (Firestore/Auth/Hosting/Functions/Storage), Anthropic Claude API, Stripe (ยังไม่ต่อ), LINE Official (ยังไม่ต่อ), Google Maps API, ผู้ให้บริการโดเมน, GitHub — อัปเดตทุกครั้งที่เพิ่มการเชื่อมต่อใหม่

9. วงจรชีวิตประกาศ & การลบข้อมูล/รูปภาพอัตโนมัติ

นับถอยหลังอัตโนมัติจากวันที่มีอยู่แล้ว → แจ้งเตือนลูกค้าเป็นชั้น (30 วัน → 7 วัน → ถึงกำหนด) → ลูกค้ากดปุ่มเดียว "ขอเก็บไว้ต่อ" ได้ → ถึงกำหนดจริงลบเฉพาะรูปจาก Storage คงข้อมูลข้อความไว้ → แอดมินเห็นแค่สรุปในแดชบอร์ด ไม่ต้องลงมือทำเอง

10. ระบบจัดการแบนเนอร์ (Admin Backend)

CRUD เต็มรูปแบบ ไม่ต้องแก้โค้ด — รูป/ลิงก์/ตำแหน่ง+หน้า/วันเริ่ม-หมดอายุ/สถานะ — เริ่มด้วยเนื้อหาของเจ้าของเว็บเองกันหน้าว่าง แล้วถอด/ลดสัดส่วนได้ทันทีเมื่อมีลูกค้าจริง

11. ลำดับการสร้างจริง — สถานะปัจจุบัน

แบ่งเป็น 2 ชุดตามที่ต้องพึ่งอะไรจากคุณหรือไม่ — สีบอกสถานะแต่ละข้อ
ชุด A — ไม่ต้องรอเตรียมอะไรเพิ่ม

✓

12. แผนเปิดตัว (Go-to-Market)

หลักการรวม: ทุกกลุ่ม (ผู้ซื้อทั่วไป/เจ้าของบ้าน/Agent) ใช้ฟรีก่อนเสมอ แล้วค่อยเปิดจ่ายทีละชั้นเมื่อเห็นผลจริง — ผู้ซื้อ/ผู้เช่าใช้ฟรีตลอดไป (แต่เก็บข้อมูลความสนใจอย่างเป็นระบบเสมอ), เจ้าของบ้าน(ขาย+เช่า)โพสต์ฟรีก่อนแล้วค่อยเสนอ VIP, Agent โพสต์ฟรีช่วง Founding แล้วค่อยเก็บค่าสมาชิก+เสนอ VIP

  1. ปิดเงียบ (Soft launch) 2-4 สัปดาห์ — เติมประกาศ 50+ รายการ
  2. เปิดวงในพื้นที่ + เปิดรับ "Founding Agents" ฟรี
  3. SEO/Google Business คู่ขนาน
  4. เปิดเก็บเงินทาง 2 เมื่อ traffic/lead สม่ำเสมอ
  5. เปิดทาง 3 (โฆษณาลูกค้าภายนอก) เมื่อมีตัวเลข traffic
  6. เปิดทาง 4 (แพ็กเกจ VIP ทั้งสองฝั่ง) เมื่อฐาน Agent/พูลมั่นคง

13. หมายเหตุค่าใช้จ่าย

Firebase อยู่ใน free tier ได้นานในสเกลนี้ ค่าใช้จ่ายเพิ่มตามการใช้งานจริงคือ Claude API (ต่อข้อความ) และ Stripe (~3-4% ต่อธุรกรรม) — ไม่มีค่า license ก้อนใหญ่ผูกพันรายเดือน
