## แก้ไข Username/Password ทีหลัง
เข้า Admin Dashboard → คลิก 🖼️ Site Content → เลื่อนลงล่างสุด หัวข้อ "Admin Login Details" → แก้ username/password (และอีเมล/เบอร์สำรองถ้าต้องการ) → กด Save

## อัปเดตในรอบนี้
- ทุกหน้า Admin (Dashboard, Map, Contacts, Site Content) ตอนนี้ต้องล็อกอินก่อนถึงจะเข้าได้จริง — ไม่ใช่แค่หน้าตาเหมือนเดิม
- ใส่ user/pass ผิดจะมีข้อความแจ้งเตือน ใส่ถูกจะเข้าสู่ระบบและจำไว้จนกว่าจะปิดแท็บ/กด Log Out
- ทดสอบแล้ว: ล็อกอินผิด ถูก, ล็อกเอาท์, และเข้าหน้า Admin ตรงๆโดยไม่ล็อกอินจะถูกเด้งไปหน้า login อัตโนมัติ

## ⚠️ ข้อควรรู้สำคัญ
ระบบนี้เป็นการป้องกันแบบเบื้องต้น (เก็บ user/pass ไว้ใน Firestore) **ไม่ใช่ระบบความปลอดภัยระดับที่ใช้ปกป้องข้อมูลจริงจังได้ 100%** — คนที่เปิด devtools ดูโค้ดเว็บสามารถเห็นวิธีการทำงานได้ ก่อนเปิดใช้งานจริงกับข้อมูลลูกค้า/เจ้าของบ้านจริง แนะนำให้อัปเกรดเป็น Firebase Authentication (ระบบล็อกอินมาตรฐานของ Firebase เอง) — สามารถแจ้งนักพัฒนาที่มาช่วยต่อยอดได้เลย
