# huahin.properties — อัปเดตล่าสุด (แก้ภาษาปนกัน — ร่างเป็นอังกฤษก่อนเสมอ)

## สาเหตุที่แท้จริงและการแก้ไข
เจอสาเหตุจริงแล้วครับ: ระบบค้นหาสถานที่ใกล้เคียง (Google Maps) ไม่ได้ระบุภาษาไว้ จึงตอบกลับเป็น**ภาษาไทยเสมอ**ไม่ว่าจะขอภาษาอะไร แล้วข้อความนั้นถูกแปะเข้าไปหลังขั้นตอนแปลภาษาเสร็จแล้ว จึงไม่ถูกแปลตามและเกิดภาษาปนกัน

**แก้ไขแล้วโดยปรับโครงสร้างทั้งระบบ:**
1. Google Maps ดึงข้อมูลเป็นภาษาอังกฤษเสมอแล้ว (ชื่อสถานที่ + ระยะทาง + เวลา)
2. AI ร่างเนื้อหาทรัพย์เป็น**ภาษาอังกฤษเป็นหลักก่อนเสมอ** (รวมรายการสถานที่ใกล้เคียงที่ผนวกเข้าไปตอนนี้)
3. จากนั้นแปลจากต้นฉบับอังกฤษเป็นไทย/รัสเซีย/จีน/เยอรมัน/นอร์เวย์ **ทั้งหมดพร้อมกัน** — ทำให้สถานที่ใกล้เคียง หน่วยวัด (กม./นาที) และทุกประโยคถูกแปลอย่างถูกต้องสม่ำเสมอในทุกภาษา ไม่มีภาษาปนกันอีก
4. หน้าตรวจสอบร่าง (ที่แอดมินเห็น) ยังคงแสดงเป็น**ภาษาไทย**เหมือนเดิม (แปลมาจากอังกฤษ) เพื่อให้ตรวจสอบได้สะดวก

## วิธีใช้งาน
1. อัปโหลดไฟล์ทั้งหมดในโฟลเดอร์นี้ขึ้น repo บน GitHub ที่ระดับ root (ทับไฟล์เดิมทั้งหมด)
2. รอ GitHub Pages deploy เสร็จ (1-2 นาที) แล้ว Hard Refresh (Ctrl+Shift+R)
3. ทดสอบเพิ่มทรัพย์ใหม่พร้อมพิกัดแผนที่ แล้วเช็คว่าคำบรรยายภาษาไทย/อังกฤษ/จีน ฯลฯ ไม่มีคำปนกันแล้ว
