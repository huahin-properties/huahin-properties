1. บทสรุปผู้บริหาร

huahin.properties คือเว็บมาร์เก็ตเพลสอสังหาริมทรัพย์เฉพาะพื้นที่หัวหิน/ปราณบุรี/ชะอำ ประกอบด้วยเว็บสาธารณะสำหรับผู้ซื้อ/ผู้เช่า, ระบบให้เจ้าของบ้าน/นายหน้าสมัครสมาชิกและลงประกาศเอง (self-serve), ผู้ช่วย AI ตอบลูกค้าอัตโนมัติ, และหลังบ้านสำหรับแอดมินจัดการทุกอย่าง ระบบทั้งหมดสร้างด้วยสถาปัตยกรรม "Design Component" (หน้าเว็บอิสระต่อไฟล์) เชื่อมกับฐานข้อมูล Firebase เพียงชุดเดียว ทำให้ทุกหน้าเห็นข้อมูลตรงกันแบบเรียลไทม์

รายได้ของธุรกิจมาจาก 3 ทางหลัก: (1) ค่าคอมมิชชันจากลีดที่ระบบกรองมาให้, (2) ค่าสมาชิกรายเดือนจากเจ้าของบ้าน/นายหน้าที่ต้องการลงประกาศเอง (พร้อมระบบทดลองใช้ฟรี), (3) ค่าโฆษณา/แบนเนอร์ ทั้งหมดเก็บเงินผ่าน Stripe อัตโนมัติ

2. สถาปัตยกรรมเทคนิคโดยรวม

ระบบแบ่งเป็น 4 ชั้นทำงานร่วมกัน:

ชั้นเทคโนโลยีหน้าที่
หน้าเว็บ (Frontend)Design Components (ไฟล์ .dc.html แยกต่อหน้า)หน้าเว็บสาธารณะ + หน้าสมาชิก + หน้าแอดมิน ทั้งหมดวาดผลทันทีที่โหลด
ฐานข้อมูลFirebase Firestoreเก็บทรัพย์ สมาชิก เจ้าของ/ผู้เช่า ลีด การชำระเงิน ฯลฯ ทุกหน้าอ่าน/เขียนชุดเดียวกัน
ไฟล์รูปภาพFirebase Storageเก็บรูปทรัพย์/โปรไฟล์จริง (ย้ายจากการฝังในฐานข้อมูลมาไว้ที่นี่แล้ว)
ประมวลผลฝั่งเซิร์ฟเวอร์Firebase Cloud Functionsตัวกลางเรียก Claude AI (กัน API key รั่ว) + รับ Webhook จาก Stripe

บริการภายนอกที่เชื่อมต่อ: Anthropic Claude API (สมองแชทบอท + ผู้ช่วยร่างประกาศ/แปลภาษา), Stripe (รับชำระเงินสมาชิก/บูสต์/โฆษณา), Google Maps JavaScript API (แผนที่ปักหมุดทรัพย์), GitHub (ที่เก็บโค้ด ใช้อัปโหลดไฟล์เวลาอัปเดต ไม่ใช่ระบบที่เว็บเรียกใช้งานตอนทำงานจริง)

การขึ้นเว็บจริงใช้ Firebase Hosting — ไม่มีเซิร์ฟเวอร์ที่ต้องดูแลเอง ปรับขนาดตามผู้ใช้งานอัตโนมัติ

3. หน้าเว็บทั้งหมดในระบบ

3.1 ฝั่งสาธารณะ (ผู้ซื้อ/ผู้เช่าทั่วไป)

  • หน้าแรก (Home) — Hero, ช่องค้นหา, ปุ่ม CTA 4 ปุ่ม, กริดทรัพย์แนะนำ (พร้อม Featured/แบนเนอร์ปนอยู่ด้านบนสุด), แบ่งหน้า 10 รายการ/หน้า
  • ผลการค้นหา (Search Results) — กริด/รายการ พร้อมตัวกรอง (โซน/ประเภท/ราคา/ห้องนอน/สถานะ ฯลฯ) และการเรียงลำดับ Featured ก่อนเสมอ
  • รายละเอียดทรัพย์ (Property Details) — แกลเลอรีรูป, ข้อมูลครบ, ระยะทางจุดสำคัญใกล้เคียง, ทรัพย์ใกล้เคียง, ปุ่มติดต่อ (โทร/WhatsApp/LINE/อีเมล), ฟอร์มสอบถาม, นับจำนวนผู้เข้าชม
  • ฝากขาย/เช่า (Sell) — ฟอร์มให้เจ้าของบ้านกรอกข้อมูลทรัพย์เบื้องต้นส่งเข้าระบบ
  • เกี่ยวกับเรา / ติดต่อเรา (About / Contact)
  • โฆษณา (Advertise) — ธุรกิจภายนอกซื้อพื้นที่แบนเนอร์เอง จ่ายผ่าน Stripe ทันที
  • AI Concierge — หน้าทางเลือกที่เปิดมาเจอกล่องคุยกับ AI เต็มจอ ใช้ทำการตลาดแยกจากหน้าแรก

3.2 ฝั่งสมาชิก (เจ้าของบ้าน/นายหน้าที่ลงประกาศเอง)

  • สมัคร/เข้าสู่ระบบ (Agent Signup) — สมัครฟรีทันที ไม่ต้องรอแอดมินอนุมัติบัญชี (อนุมัติเฉพาะ "ประกาศ" แต่ละรายการ ไม่ใช่ตัวบัญชี)
  • หน้าทำงานของฉัน (Lister Dashboard) — ศูนย์กลางของสมาชิก: รายการทรัพย์ของตัวเอง + สถานะ (ร่าง/รออนุมัติ/กำลังแสดง/หมดอายุ/ถูกปฏิเสธ) + วันหมดอายุ + จำนวนผู้เข้าชม, ฟอร์มเพิ่ม/แก้ไขทรัพย์, ตั้งค่าเว็บไซต์ส่วนตัว (ภาพปก/สีธีม/ลิงก์โซเชียล/ข้อความแนะนำตัว), เครื่องมือร่างข้อความโพสต์ Facebook ด้วย AI, ระบบลงทะเบียนผู้ซื้อ (เฉพาะสมาชิก VIP)
  • แพ็กเกจ & ราคา (Lister Billing) — ตารางเทียบแพ็กเกจ, ปุ่มเริ่มทดลองใช้ฟรี/อัปเกรดที่พาไปชำระเงินผ่าน Stripe จริง, ส่วนสมัคร VIP nayหน้า (ปิดซ่อนอยู่จนกว่าจะปลดล็อกตามจำนวนสมาชิก)
  • เว็บไซต์ส่วนตัวของสมาชิก (Agent Profile) — หน้าสาธารณะรวมทรัพย์ทั้งหมดของสมาชิกคนหนึ่ง แชร์ให้ลูกค้าดูได้โดยตรง ปรับแต่งได้ตามระดับแพ็กเกจ

3.3 ฝั่งแอดมิน (เจ้าของเว็บ/ทีมงาน)

  • เข้าสู่ระบบแอดมิน (Admin Login)
  • แดชบอร์ดแอดมิน (Admin Dashboard) — สรุปสถิติทรัพย์ทุกสถานะ, ตัวติดตามสัญญาเช่าใกล้หมดอายุ, การ์ดแจ้งเตือน "สถานะระบบ" 3 หมวด (เช็คอัตโนมัติ/ปฏิทินความปลอดภัย/บริการภายนอก), ปุ่มตั้งค่า Featured/VIP boost ต่อทรัพย์, ลิงก์เข้าเครื่องมือย่อยทั้งหมด
  • คิวอนุมัติประกาศ (Listing Approvals) — ตรวจ/อนุมัติ/ปฏิเสธประกาศใหม่จากสมาชิก พร้อมแจ้งเตือนบัญชีต้องสงสัยว่าสมัครทดลองซ้ำ (จับคู่เบอร์โทร)
  • รายชื่อสมัครสมาชิก (Agent Approvals) — ดู/ระงับบัญชีสมาชิกที่ประพฤติไม่เหมาะสม
  • แผนที่ทรัพย์ (Property Map) — Google Maps ปักหมุดทุกทรัพย์ สีตามสถานะ/ประเภท คลิกดู/แก้ไขได้
  • ไดเรกทอรีรวม (Owners) — เจ้าของ/ผู้เช่า/ผู้ซื้อในที่เดียว เชื่อมโยงกับรหัสทรัพย์
  • เพิ่มทรัพย์ด่วนด้วย AI (AI Quick Add) — แอดมินถ่ายรูป/พูดบอกข้อมูล ให้ Claude AI ร่างประกาศ 8 ภาษา + คำนวณระยะทางจุดสำคัญ + โพสต์ Facebook อัตโนมัติ
  • ตั้งค่าเนื้อหาเว็บ (Site Content) — ข้อความ Facebook post, บุคลิก/ข้อจำกัดของแชทบอท AI, ราคาแบนเนอร์/Featured/VIP, สวิตช์เปิด-ปิดฟีเจอร์ (Rollout Level), เปลี่ยนรหัสผ่านแอดมิน
  • ทีมงาน (Team / Staff Signup) — เชิญ/จัดการบัญชี Staff แยกสิทธิ์จาก Owner

4. ระบบสมาชิกและแพ็กเกจ (Membership v2)

แทนที่แนวคิด "แพ็กเกจฟรีถาวร" เดิมทั้งหมด ด้วยโครงสร้างที่จูงใจให้ทดลองแล้วอัปเกรดจริง:

ระดับราคา/เดือนจำนวนประกาศตัวเร่งการมองเห็น
🎁 ทดลองใช้ฟรี (ครั้งเดียว 30 วัน)ฟรี5เทียบเท่าแพ็กเกจ 3 ทุกอย่าง
แพ็กเกจ 1590 บาท5ลิงก์โซเชียล 1 ช่องทาง
แพ็กเกจ 21,190 บาท12Featured 2 รายการ + ลิงก์โซเชียล 2 ช่องทาง
แพ็กเกจ 32,290 บาท25Featured ไม่จำกัด + ลิงก์โซเชียลครบ 4 ช่องทาง
ระดับ 4-5 (VIP เงิน/ทอง/เพชร)เตรียมไว้/ยังไม่เปิด—เข้าพูล Agent VIP ช่วยขาย (ปลดล็อกทีหลังเมื่อฐานสมาชิกพอ)

หลักการ: ฟีเจอร์พื้นฐานทุกแพ็กเกจเท่ากัน (แปล 8 ภาษา, ข้อมูลติดต่อ, สถิติวิว, แก้ไขประกาศ, ออกแบบเว็บไซต์/แก้ไขโปรไฟล์) — สิ่งที่ต่างกันมีแค่ จำนวนประกาศ และ ตัวเร่งการมองเห็น ฟีเจอร์ที่แพ็กเกจนั้นยังไม่มีสิทธิ์จะแสดงเป็นสีเทา (ไอคอนล็อก) ไม่ใช่ซ่อนหายไปเฉยๆ เพื่อกระตุ้นให้อัปเกรด

การใช้ trial ครั้งเดียวต่อคน (ล็อกถาวรใน field ของบัญชี ใช้แล้วเริ่มใหม่ไม่ได้อีก) และมีระบบตรวจจับสมัครซ้ำโดยจับคู่เบอร์โทรศัพท์ให้แอดมินตรวจสอบในคิวอนุมัติ

5. วงจรชีวิตของประกาศ

ทุกประกาศมีสถานะ 4 แบบ ไหลเป็นลำดับ:

⏳ รออนุมัติ
→
✅ กำลังแสดง (30 วัน)
→
⏰ หมดอายุ (ซ่อนจากเว็บ)
→ กดต่ออายุ →
✅ กลับมาแสดงทันที

รายละเอียดสำคัญ:

  • ประกาศใหม่ทุกรายการต้องผ่านแอดมินอนุมัติก่อนขึ้นเว็บจริงเสมอ
  • ครบ 30 วันนับจากวันอนุมัติ → ซ่อนจากหน้าเว็บสาธารณะอัตโนมัติ แต่ข้อมูลไม่ถูกลบ
  • เจ้าของกด "ต่ออายุ" ได้เองทันที ไม่ต้องรอแอดมินอนุมัติซ้ำ
  • ถ้าปฏิเสธ แอดมินระบุเหตุผลได้ เจ้าของแก้ไขแล้วส่งใหม่ได้
  • ประกาศที่หมดอายุเกิน 90 วันไม่มีการต่ออายุ → ระบบลบเฉพาะไฟล์รูป (เก็บรูปปก 1 รูป + ข้อความ/ราคาไว้ตลอดไป) เพื่อประหยัดพื้นที่จัดเก็บ
  • นับจำนวนผู้เข้าชมทุกประกาศ แสดงให้เจ้าของเห็นในหน้าทำงานของตัวเอง ใกล้หมด trial จะสรุปยอดรวมพร้อมชวนอัปเกรด

⚠️ ข้อจำกัดทางเทคนิค: การตรวจสอบวันหมดอายุ/ลบรูป ทำงานแบบ "สแกนตอนมีคนเปิดหน้าแดชบอร์ด" ไม่ใช่ระบบตั้งเวลาอัตโนมัติในเซิร์ฟเวอร์ (ยังไม่ได้ติดตั้ง Cloud Scheduler) หมายความว่าถ้าไม่มีใครเข้าเว็บเลยเกิน 30 วัน การซ่อน/ลบจะล่าช้าไปจนกว่าจะมีคนเข้าใช้อีกครั้ง

6. ระบบ AI

6.1 แชทบอทลูกค้า (ContactRail — ทุกหน้าเว็บสาธารณะ)

  • ตอบคำถามทั่วไป/ทักทายแบบธรรมชาติ + ตอบข้อมูลจริง (ราคา/ห้อง/สถานะ/รหัสทรัพย์) จากฐานข้อมูลจริงเท่านั้น ไม่มโนข้อมูล
  • ปฏิเสธหัวข้อต้องห้าม (ตั้งค่าได้จากหลังบ้าน) อย่างสุภาพ ไม่เปิดเผยว่ามีข้อจำกัดอยู่
  • ตรวจจับภาษาลูกค้าอัตโนมัติ ตอบครบทั้ง 8 ภาษา
  • เจอรหัสทรัพย์ในคำตอบ → สร้างปุ่ม "ดูรายละเอียด" อัตโนมัติ, เจอคำว่า "ติดต่อเรา" → สร้างปุ่มเปิดฟอร์มสอบถาม
  • บุคลิก (ชื่อ/บทบาท) และหัวข้อต้องห้ามแก้ไขได้จากหน้า Site Content โดยไม่ต้องแก้โค้ด
  • รู้บริบทหน้าที่ลูกค้ากำลังเปิดดูอยู่ (ผูกรหัสทรัพย์อัตโนมัติ) และจำกัดจำนวนทรัพย์ที่ส่งให้ AI อ่านต่อข้อความ (คุมต้นทุนเมื่อทรัพย์เยอะขึ้น)
  • บันทึกบทสนทนาไว้ในเบราว์เซอร์ลูกค้า คุยต่อได้ข้ามหน้า ไม่เริ่มใหม่ทุกครั้ง

6.2 AI Quick Add (เครื่องมือแอดมิน)

แอดมินถ่ายรูปหลายรูป + พูดบอกข้อมูล (voice dictation) + ปักหมุด Google Maps → Claude AI ร่างประกาศให้ครบ 8 ภาษา พร้อมคำนวณระยะทาง 10+ จุดสำคัญใกล้เคียง และร่างโพสต์ Facebook พร้อมแฮชแท็ก แอดมินตรวจ/แก้ก่อนบันทึกจริง

6.3 แปลภาษาอัตโนมัติ (สมาชิก self-serve)

เมื่อสมาชิกกรอกคำอธิบายทรัพย์/ข้อความแนะนำตัวเป็นภาษาเดียวแล้วกดบันทึก ระบบยิง AI แปลเป็นครบ 8 ภาษาให้อัตโนมัติทันที (เรียกครั้งเดียวตอนบันทึก ไม่ใช่แปลสดทุกครั้งที่มีคนเปิดดู ประหยัดต้นทุน)

7. ระบบการเงิน (Stripe)

ทุกรายการเงินผ่าน Stripe Checkout จริง (โหมด Test พร้อมเปลี่ยนเป็น Live ได้) มี Webhook ยืนยันลายเซ็นก่อนอัปเดตสถานะเสมอ กันข้อมูลปลอม:

  • ค่าสมาชิกรายเดือน (แพ็กเกจ 1/2/3) — subscription ต่อเนื่อง จ่ายไม่ผ่าน/ยกเลิก → ซ่อนประกาศอัตโนมัติ ไม่ลบข้อมูล
  • Featured Listing boost — จ่ายครั้งเดียว 7/30 วัน ดันประกาศขึ้นก่อน
  • VIP boost ฝั่งเจ้าของบ้าน (เงิน/ทอง/เพชร) — จ่ายครั้งเดียว/30 วัน เข้าพูลให้ Agent VIP เห็น
  • Agent VIP subscription — ฝั่งนายหน้าสมัครสมาชิก VIP รายเดือน (รอเปิดใช้งานเมื่อฐานสมาชิกมากพอ)
  • แบนเนอร์โฆษณา — ธุรกิจภายนอกจ่ายครั้งเดียว 30 วันต่อตำแหน่ง
  • ทุกการจ่ายเงินสำเร็จถูกบันทึกลง Ledger (`payments`) เขียนได้เฉพาะผ่านระบบเซิร์ฟเวอร์เท่านั้น ตรวจสอบย้อนหลังได้

ปุ่มจัดการ/ยกเลิกการเป็นสมาชิกใช้ Stripe Customer Portal สำเร็จรูป ไม่ต้องสร้างหน้าเอง

8. ความปลอดภัยและสิทธิ์การเข้าถึง

  • Firebase Authentication จริง (ไม่ใช่รหัสผ่านฝังโค้ดแบบเดิม) แยกบัญชี Owner / Staff / สมาชิกลงประกาศ / ผู้ใช้ทั่วไป
  • Firestore Security Rules แบบ role-based + deny-by-default — สมาชิกแก้ไข/ลบได้เฉพาะทรัพย์ของตัวเอง, ห้ามตั้งสถานะ "กำลังแสดง" เอง (ต้องผ่านแอดมินอนุมัติ ยกเว้นการต่ออายุ), แก้ไขได้เฉพาะฟิลด์ที่กำหนด
  • สิทธิ์ 2 ระดับในทีมงาน: Owner เห็น/แก้ทุกอย่างรวมการเงิน, Staff จัดการประกาศ/ลีด/แชทได้แต่แตะการเงิน-ตั้งค่าไซต์ไม่ได้
  • ฟิลด์สำคัญของทรัพย์ (รูปปก/ราคา/ที่อยู่/รหัส) ถูกล็อกถาวรเมื่อขายสำเร็จ กันการ "สวมประกาศ" เอาทรัพย์ใหม่มาแทนของเดิม
  • API key ของ Claude เก็บเป็นความลับฝั่งเซิร์ฟเวอร์เท่านั้น ไม่มีทางรั่วออกฝั่งเบราว์เซอร์ผู้ใช้
  • Stripe Webhook ตรวจลายเซ็นทุกครั้งก่อนเชื่อ กันคำสั่งปลอม

⚠️ ยังไม่ได้ทำ: Firebase App Check (กันบอทยิง request ตรง), Rate limiting ต่อ IP, reCAPTCHA บนฟอร์ม — เป็นงานเสริมด้านความปลอดภัยที่ยังไม่เร่งด่วนแต่ควรทำก่อนสเกลใหญ่

9. SEO

  • URL ถาวรต่อประกาศ + Meta title/description เฉพาะหน้า + Open Graph สำหรับแชร์โซเชียล
  • Structured data (schema.org RealEstateListing) แบบ JSON-LD ทุกหน้าทรัพย์ — ช่วย Google เข้าใจว่าเป็นประกาศอสังหาฯ
  • Sitemap.xml + robots.txt ยื่น Google Search Console สำเร็จแล้ว (ครอบคลุมหน้า static หลัก — ยังไม่ครอบคลุมหน้าทรัพย์รายตัวที่สร้างจาก Firestore สด)
  • ลิงก์กลับเว็บ (backlink) ฝังอัตโนมัติในทุกโพสต์ Facebook ที่ AI ร่างให้ + ลิงก์แชร์ที่สมาชิกกดคัดลอกได้เอง

10. โครงสร้างข้อมูลหลัก (แนวคิด ไม่ใช่โค้ด)

ฐานข้อมูลแบ่งเป็น "คอลเลกชัน" (เหมือนตารางในฐานข้อมูลทั่วไป) หลักๆ ดังนี้:

คอลเลกชันเก็บอะไร
propertiesทรัพย์ทุกรายการ — ข้อมูล+สถานะวงจรชีวิต+วันหมดอายุ+จำนวนวิว
propertyPhotosลิงก์รูปภาพแต่ละใบ (ไฟล์จริงอยู่ Storage)
listersบัญชีสมาชิกที่ลงประกาศเอง + แพ็กเกจ/trial/VIP ของแต่ละคน
owners / tenantsไดเรกทอรีเจ้าของบ้าน/ผู้เช่า (ที่แอดมินดูแลเอง)
leads / listerLeadsผู้สนใจที่ทักเข้ามาผ่านฟอร์ม/แชท
buyerRegistrationsการลงทะเบียนผู้ซื้อของ Agent VIP (กันข้อพิพาท)
paymentsประวัติการจ่ายเงินทุกรายการจาก Stripe
bannersแบนเนอร์โฆษณาทุกตำแหน่ง
siteContent / settingsค่าตั้งค่าทั้งหมดที่แอดมินแก้ได้เอง (ราคา/บุคลิก AI/ข้อความ/สวิตช์เปิดปิดฟีเจอร์)
adminUsers / staffInvitesบัญชีทีมงานและคำเชิญ

11. กระบวนการอัปเดตเว็บ (Deployment)

เจ้าของเว็บไม่ใช้เครื่องมือพัฒนาในเครื่องตัวเอง (ไม่มี git/command line ส่วนตัว) ขั้นตอนอัปเดตทุกครั้ง:

  1. ทีมออกแบบ/พัฒนาเตรียมไฟล์ที่แก้ไขไว้ให้ดาวน์โหลดเป็นชุด
  2. เจ้าของเว็บอัปโหลดไฟล์ทับใน GitHub ผ่านหน้าเว็บ (Add file → Upload files) — ไม่ใช้คำสั่ง git
  3. เว็บไซต์หลัก (Hosting) อัปเดตอัตโนมัติหลังอัปโหลด
  4. ถ้ามีการแก้ Cloud Functions หรือ Security Rules ต้องเปิด GitHub Codespace (เทอร์มินัลบนคลาวด์) แล้วรันคำสั่ง `firebase deploy` เพิ่มอีกขั้นหนึ่ง

มีระบบ "ลิงก์ทดลอง" (Firebase Hosting Preview Channels) ให้ทดสอบฟีเจอร์ใหม่ก่อนขึ้นเว็บจริงเสมอ กันเว็บจริงพังกลางอากาศระหว่างพัฒนา
