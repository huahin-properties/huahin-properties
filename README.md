# huahin.properties — ไฟล์เว็บสำหรับอัปโหลดขึ้น GitHub

## วิธีใช้งาน
1. อัปโหลดไฟล์ทั้งหมดในโฟลเดอร์นี้ (รวมทั้งไฟล์ `.image-slots.state.json` ที่ซ่อนอยู่) ขึ้น repo ของคุณ ที่ระดับ root
2. เปิด GitHub Pages (Settings → Pages → source: main branch, root)
3. ผูกโดเมนจาก GoDaddy ตามที่แนะนำในแชท
4. หน้าแรกของเว็บคือ `index.html` (สำเนาของ Home.dc.html)

## ⚠️ ข้อจำกัดสำคัญที่ต้องรู้ก่อนใช้งานจริง

**1. ข้อมูลทรัพย์/เจ้าของ/ผู้เช่า (data.js + localStorage)**
- ข้อมูลตัวอย่าง 21 รายการอยู่ใน `data.js` — เป็นข้อมูลเริ่มต้นที่ทุกคนเห็นเหมือนกัน
- แต่เมื่อ Admin แก้ไข/เพิ่ม/ลบทรัพย์ผ่านหน้า Admin Dashboard จริง การเปลี่ยนแปลงจะถูกบันทึกใน **localStorage ของเบราว์เซอร์คนนั้นคนเดียว** เท่านั้น — คนอื่นที่เข้าเว็บจะยังเห็นข้อมูลเดิมใน data.js
- นี่ใช้ได้สำหรับทดสอบ/เดโม แต่ใช้งานจริงหลายคนพร้อมกันไม่ได้ ต้องมีฐานข้อมูลกลาง (แนะนำ Firebase/Supabase) ก่อนเปิดใช้จริง

**2. รูปภาพ (.image-slots.state.json)**
- รูปทั้งหมดที่อัปโหลดไว้ในต้นแบบนี้ถูกเก็บในไฟล์ `.image-slots.state.json` ไฟล์เดียว (เป็น base64 ในตัวไฟล์เอง)
- วิธีนี้ใช้ได้กับรูปจำนวนไม่มาก แต่จะช้าลงเรื่อยๆเมื่อไฟล์โตขึ้น และไม่รองรับหลายคนอัปโหลดพร้อมกัน
- **ก่อนใช้งานจริง แนะนำให้ย้ายไปใช้ Cloudinary** (ฟรี, มี CDN, โหลดเร็ว) — ต้องแก้โค้ดส่วนอัปโหลดรูปใน `image-slot.js` และหน้า Admin ให้เรียก Cloudinary API แทน

## สรุป
ไฟล์ชุดนี้เหมาะสำหรับ**ดูตัวอย่าง/เดโม/ทดสอบ**บนโดเมนจริงได้ทันที แต่ก่อนเปิดให้ลูกค้าใช้งานจริง ควรต่อฐานข้อมูลกลางและ Cloudinary ตามข้อ 1-2 ข้างต้น — แนะนำให้ส่งไฟล์ชุดนี้พร้อมข้อมูลนี้ให้นักพัฒนาช่วยต่อยอด
