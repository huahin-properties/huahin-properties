# huahin.properties — ไฟล์เว็บสำหรับอัปโหลดขึ้น GitHub

## วิธีใช้งาน
1. อัปโหลดไฟล์ทั้งหมดในโฟลเดอร์นี้ขึ้น repo ของคุณ ที่ระดับ root (ทับไฟล์เดิมได้เลย ไม่ต้องลบก่อน)
2. เปิด GitHub Pages (Settings → Pages → source: main branch, root)
3. ผูกโดเมนจาก GoDaddy ตามที่แนะนำในแชท
4. หน้าแรกของเว็บคือ `index.html` (สำเนาของ Home.dc.html)

## ✅ สถานะปัจจุบัน — ใช้ฐานข้อมูลกลางจริงแล้ว (Firebase)

ต่างจากไฟล์ชุดแรกที่เคยส่งให้ — ตอนนี้เว็บทุกหน้าอ่าน/เขียนข้อมูลจาก **Firebase (Firestore)** ซึ่งเป็นฐานข้อมูลกลางบนคลาวด์ ไม่ใช่ localStorage ของเบราว์เซอร์แต่ละคนอีกต่อไป หมายความว่า:

- ข้อมูลทรัพย์ 24 รายการ, เจ้าของ, ผู้เช่า — ทุกคนที่เข้าเว็บเห็นข้อมูลชุดเดียวกัน
- Admin แก้ไข/เพิ่ม/ลบทรัพย์ผ่าน Admin Dashboard → เปลี่ยนแปลงจริงในฐานข้อมูลกลางทันที ทุกคนเห็นการเปลี่ยนแปลงเมื่อรีเฟรช
- รูปภาพที่อัปโหลด (ทั้งรูปทรัพย์และรูปอ้างอิงเจ้าของ/ผู้เช่า) เก็บใน Firestore เช่นกัน ไม่หายเมื่อรีเฟรชหรือเปลี่ยนเครื่อง

การเชื่อมต่อ Firebase อยู่ในไฟล์ `firebase-client.js` — ใช้ config ของโปรเจกต์ Firebase ที่คุณสร้างไว้แล้ว (`huahin-properties-5f1b5`) ไม่ต้องตั้งค่าอะไรเพิ่มตอนอัปโหลดขึ้น GitHub

## ⚠️ ข้อควรรู้ก่อนใช้งานจริงกับลูกค้า

**1. Firestore Security Rules ตอนนี้เปิดกว้างชั่วคราว (test mode)** — หมดอายุอัตโนมัติ 8 ส.ค. 2026 ตามที่ Firebase Console กำหนด ก่อนถึงวันนั้นควรกลับไปตั้งกฎที่รัดกุมกว่านี้ (เช่น อนุญาตอ่านทุกคนแต่จำกัดการเขียนเฉพาะ Admin ที่ login แล้ว) — ควรปรึกษาเรื่องนี้กับนักพัฒนาก่อนเปิดใช้งานจริงกับสาธารณะ

**2. Admin Login ปัจจุบันเป็นการเข้าสู่ระบบแบบจำลอง (mock)** ยังไม่ได้ผูกกับระบบยืนยันตัวตนจริง (เช่น Firebase Authentication) — ควรทำเพิ่มก่อนใช้งานจริง เพื่อป้องกันไม่ให้คนนอกเข้าไปแก้ไขข้อมูลทรัพย์ได้

**3. โควตาฟรีของ Firebase (Spark Plan)** เพียงพอสำหรับเว็บขนาดนี้ในช่วงเริ่มต้น (มีลิมิตการอ่าน/เขียนต่อวัน) — ถ้าปริมาณผู้เข้าชมสูงขึ้นมากในอนาคตอาจต้องพิจารณาอัปเกรดแผน

## สรุป
ไฟล์ชุดนี้พร้อมใช้งานจริงบนโดเมนของคุณได้เลย ข้อมูลและรูปภาพจะเก็บถาวรและแชร์ร่วมกันทุกคนที่เข้าเว็บ — แนะนำให้แก้ไข 2 ข้อด้านบน (security rules + real login) ก่อนเปิดให้สาธารณะใช้งานเต็มรูปแบบ
