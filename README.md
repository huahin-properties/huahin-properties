# huahin.properties — ไฟล์เว็บล่าสุด (แก้บั๊กปุ่มไมค์ + ข้อความ AI ชัดเจนขึ้น)

## แก้ไขล่าสุด
- แก้บั๊กปุ่มไมค์ "recognition has already started" ที่ขึ้นเวลากดค้างพูดต่อรอบสอง — ตอนนี้กดพูดต่อได้ลื่นแล้ว
- ปุ่ม "ให้ AI ร่างประกาศ" / "ส่งคำสั่งแก้ไขให้ AI" ตอนนี้ขึ้นข้อความอธิบายชัดเจนแทน error ทั่วไป เมื่อรันนอกระบบ claude.ai (ดูหมายเหตุสำคัญด้านล่าง — เป็นข้อจำกัดของฟีเจอร์นี้ ไม่ใช่บั๊ก)

## อัปเดตก่อนหน้า (สะสมมาในไฟล์ชุดนี้)
- หน้า "เพิ่มทรัพย์ด่วนด้วย AI" — อัปโหลดรูป+เอกสารโครงการ (PDF/รูปได้หลายไฟล์ ลบ/แก้ไขได้) + พูด/พิมพ์อธิบาย (กดไมค์ค้างพูด) → AI ร่างประกาศเป็นภาษาไทยให้ตรวจสอบ → พูด/พิมพ์แก้ไขเพิ่มได้จนพอใจ → เผยแพร่แปลครบ 6 ภาษาอัตโนมัติ พร้อมตั้งชื่อไฟล์รูปแบบ SEO
- ระบบล็อกอิน Admin จริง — Username/Password ชั่วคราว: `132435` / `qewret` (เปลี่ยนได้ที่หน้า Site Content ในแดชบอร์ด)
- แก้บั๊กรูปภาพขาดหายจากการตัดสตริงที่เครื่องหมาย `;` — แก้ครบทุกจุดแล้ว

## วิธีใช้งาน
1. อัปโหลดไฟล์ทั้งหมดในโฟลเดอร์นี้ขึ้น repo บน GitHub ที่ระดับ root (ทับไฟล์เดิมทั้งหมด)
2. รอ GitHub Pages deploy เสร็จ (1-2 นาที) แล้ว Hard Refresh (Ctrl+Shift+R)
3. เข้า `Admin Login.dc.html` ล็อกอินด้วย `132435` / `qewret`

## ⚠️ สำคัญมาก: ฟีเจอร์ "AI ร่างประกาศ" ใช้บนเว็บจริงไม่ได้ (ตามที่ทดสอบเจอ)
ฟีเจอร์นี้ใช้ `window.claude.complete` ซึ่ง**ทำงานได้เฉพาะภายในหน้าต่างพรีวิวของระบบนี้ (claude.ai) เท่านั้น** — เมื่อนำไปรันบนโดเมนจริง (huahin.properties ผ่าน GitHub Pages) จะขึ้นข้อความแจ้งว่าใช้งานไม่ได้ (ตามที่คุณเจอและแคปมาให้ดู) นี่ไม่ใช่บั๊ก แต่เป็นข้อจำกัดของฟีเจอร์นี้ที่ต้องแจ้งไว้ชัดเจน

**ถ้าต้องการให้ฟีเจอร์นี้ทำงานบนเว็บจริง** ต้องเชื่อมต่อ Claude API ของ Anthropic เอง (มี API key และเขียนโค้ดฝั่ง backend เพิ่ม) — เป็นงานพัฒนาต่อยอดที่แยกจากงานตอนนี้ ถ้าสนใจแจ้งมาได้ ผมจะแนะนำแนวทางเพิ่มเติมให้ครับ

ฟีเจอร์อื่นๆ ทั้งหมด (Firebase, ล็อกอิน, อัปโหลดรูป, ค้นหา, แผนที่ ฯลฯ) ใช้งานได้เต็มรูปแบบบนเว็บจริงทันที ไม่มีข้อจำกัดนี้
