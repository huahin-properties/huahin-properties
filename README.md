# huahin.properties — ไฟล์เว็บสำหรับอัปโหลดขึ้น GitHub (อัปเดต: แก้ปัญหารูปอัปโหลดแล้วไม่โชว์)

## สาเหตุที่พบ และการแก้ไข
เจอสาเหตุจริงแล้วครับ: รูปภาพจริงที่ถ่ายจากมือถือมีขนาดใหญ่มาก เมื่อระบบแปลงเป็นข้อมูลเก็บ (base64) แล้วมีขนาดใหญ่เกิน 200KB ต่อรูป พอหน้า Admin Dashboard ต้องแสดงรูปปกของทรัพย์ทั้ง 24 รายการพร้อมกัน ข้อมูลที่โหลดพร้อมกันเยอะเกินไปทำให้เบราว์เซอร์บางเครื่อง/บาง config ทำงานผิดพลาด รูปเลยไม่ขึ้น

**แก้ไขแล้ว:** ลดขนาดรูปที่บีบอัดตอนอัปโหลดลง (จากสูงสุด 1400px เหลือ 1000px, คุณภาพลดลงเล็กน้อย) ทำให้แต่ละรูปมีขนาดเล็กลงมาก (คาดว่าเหลือประมาณ 60-120KB ต่อรูป) เว็บจะโหลดเร็วขึ้นและเสถียรขึ้นครับ

## วิธีใช้งาน
1. อัปโหลดไฟล์ทั้งหมดในโฟลเดอร์นี้ขึ้น repo ของคุณ ที่ระดับ root (ทับไฟล์เดิมได้เลย)
2. เปิด GitHub Pages (Settings → Pages → source: main branch, root)
3. รูปเก่าที่เคยอัปโหลดไว้แล้ว (เช่น ใน CA-301) ยังอยู่ในฐานข้อมูลตามเดิม ไม่ได้ถูกลบ — แต่แนะนำให้**อัปโหลดรูปเหล่านั้นใหม่อีกครั้ง**เพื่อให้ได้ไฟล์ขนาดเล็กลงตามการแก้ไขนี้

## ทดสอบหลังอัปเดต
1. อัปโหลดไฟล์ชุดนี้ทับของเดิมใน GitHub
2. รอ 1-2 นาทีให้ GitHub Pages deploy เสร็จ
3. Hard refresh หน้าเว็บ (Ctrl+Shift+R)
4. ลองอัปโหลดรูปใหม่ที่หน้า Admin Dashboard → บันทึก → Refresh → เช็คว่ารูปยังอยู่และโชว์ในหน้า Home/Search ด้วย

- Admin Login ยังเป็นระบบจำลอง ควรผูก Firebase Authentication จริงก่อนเปิดสาธารณะ
