# HANDOVER TEST PROCEDURE — Project Handover System v1 ## 1. Test Objective Verify that a brand-new ChatGPT conversation, given only the standard handover package, can correctly understand the project (purpose, current phase, its own role, and rules) without any additional explanation from the CEO. ## 2. Required Files - `START HERE.md` - `Mission Control.dc.html` - `BLUEPRINT.md` - `HANDOVER PROMPT.md` ## 3. CEO Test Procedure 1. Open a brand-new ChatGPT conversation (no prior history). 2. Attach all four required files. 3. Paste the exact contents of `HANDOVER PROMPT.md` as the first message. 4. Send, and read the AI's response. 5. Do not answer any follow-up question from the AI by explaining the project yourself — if the AI asks something the files should have answered, that counts toward FAIL criteria (see below). 6. Record the AI's response verbatim for review. ## 4. Expected AI Behaviour Before doing anything else, the AI should: - State it has read the files in the correct order (START HERE → Mission Control → Blueprint). - Summarize the project purpose in its own words, matching Project Memory/Vision. - Correctly state the Current Phase and Current Status as recorded in Mission Control (not an invented or outdated one). - Correctly identify its own role as Product Owner (analyze/plan/review/decide PASS-FAIL — not implement). - Explicitly say it will wait for a CEO instruction before planning or implementing anything. ## 5. PASS Criteria - ☐ AI's project summary is factually consistent with Mission Control + Blueprint (no invented facts). - ☐ AI states the Current Phase correctly, matching Mission Control exactly. - ☐ AI correctly identifies its role as Product Owner and does not attempt to write code or make business decisions itself. - ☐ AI does not ask the CEO to re-explain anything already covered in the attached files. - ☐ AI ends its first response by waiting for an instruction, without starting implementation planning. ## 6. FAIL Criteria - ☐ AI states an incorrect or outdated Current Phase/Status. - ☐ AI invents information not present in any attached file. - ☐ AI asks the CEO to explain something the files already state. - ☐ AI begins implementation work or proposes a redesign before receiving an instruction. - ☐ AI misidentifies its role (e.g. acts as Lead Developer or as CEO). ## 7. Known Risks - Mission Control and Blueprint may drift out of sync over time, causing the AI to report conflicting status — the AI should flag this, not silently pick one (per START HERE's conflict rule). - If Blueprint grows very large, the AI may skim/summarize inaccurately rather than reading targeted sections tied to the current phase. - File attachment order or a missing file could cause the AI to skip a step in Read Order. ## 8. Recommended Improvements (if FAIL) - If status inconsistency is the cause: tighten Mission Control update discipline so it's updated the moment a decision/phase changes. - If the AI skims Blueprint inaccurately: consider adding a short per-phase index/table of contents to Blueprint so the AI can jump to the relevant section instead of reading linearly. - If the AI misidentifies its role: strengthen the role-definition wording in `HANDOVER PROMPT.md`. - If a file is commonly missed: add a checklist reminder to `HANDOVER PROMPT.md` confirming all four files are attached before proceeding.
