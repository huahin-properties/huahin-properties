Data Model Proposal (Revision 1.1 — ยังไม่ implement จริง)
customerSpaces/{spaceId}
  memberId, customerId|guestToken, aiThreadId, conversationId, createdAt

customerCollections/{collectionId}
  spaceId, title, icon, description, mode:"snapshot"|"live", propertyIds[], createdAt, updatedAt
  — one spaceId may have MANY collections; a property may belong to many collections across spaces

shareConversations/{spaceId}/messages/{messageId}
  from:"customer"|"member"|"ai", text, propertyContext, createdAt, readAt
  — one conversation per Space, shared across all its collections (not per-collection)

customerProfiles/{spaceId}  (Member-only, never exposed to customer)
  lookingFor, budgetRange, preferredArea, status, lastUpdatedAt
Security Model Proposal
  • customerProfiles readable only by the owning Member + authorized Admin — never by the customer's own session
  • customerCollections and shareConversations scoped by spaceId; a customer's guest token/session resolves to exactly one spaceId
  • Editing a Member's Favorites never writes into an already-created customerCollections doc — collections are copied at share time (Snapshot) or explicitly flagged Live

สถานะ: Demo UI + data model เท่านั้น — ยังไม่เชื่อม Firestore, ยังไม่มี Security Rules จริง, AI/Conversation เป็น demo static response ตาม Stop Condition รอ Product Owner review
